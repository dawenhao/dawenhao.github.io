## 1. 系统概述 XTimeline 是一套 **自定义时间轴编辑器框架**，设计目标是脱离 Unity Timeline 的 PlayableGraph 依赖，提供一套完全 **纯数据驱动**、**可序列化为 JSON** 的时间轴系统，用于游戏中的技能编辑、过场剧情、Buff 配置、控制器状态机等需要"时间轴 + 多轨道 + 片段"的场景。 ### 核心设计理念 | 理念 | 说明 | |------|------| | **纯数据驱动** | Runtime 数据层完全不依赖 Unity Engine（`noEngineReferences: true`），可独立于 Unity 运行 | | **编辑器/运行时分离** | 通过 4 个 Assembly 严格隔离编辑器逻辑与运行时数据 | | **Bridge 模式** | 通过桥接接口（Config、Json、Logger、Audio）解耦外部依赖 | | **泛型轨道模型** | 基于 `XTimelineTrackAsset` 泛型模式，一个轨道类型对应一种片段类型 | | **Attribute 驱动扩展** | 通过 `[MenuName]`、`[TrackType]`、`[TrackColor]`、`[TrackClipType]` 等特性声明式注册轨道 | | **支持多种配置类型** | Skill（`.clip`）、Director（`.director`）、Buff（`.buff`）、Story（`.story`）、Sequence（`.seq`）、Controller（`.xcontroller`）、Blueprint（`.bp`）、AI（`.ai`） | --- ## 2. 工程架构 ### 2.1 程序集（Assembly）分层
